## HTTP에 대해서 설명해주세요

HTTP (Hypertext Transfer Protocol)은 인터넷 상에서 데이터를 주고 받기 위한 **서버/클라이언트** 모델을 따르는 프로토콜입니다. 간단하게 설명하면 클라이언트와 서버가 서로 요청과 응답을 하는데 이 때 요청과 응답의 통신 표준이 http이다.

HTTP는 TCP를 사용하기 때문에 안정적인 연결이 가능한 특징이 있습니다. (주의! HTTP 3은 UDP 기반임)

HTTP의 주요 특징으로는 **무상태성**과 **비연결성**이 있습니다.

무상태성은 서버가 클라이언트의 상태를 저장하지 않는다는 특징입니다. 무상태성으로 인해 서버가 모든 클라이언트의 상태를 가지고 있지 않아도 되어 서버의 부하를 줄이는데 도움이 됩니다.

만약 클라이언트의 상태를 저장할 필요가 있다면 이 때는 쿠키 또는 세션으로 상태를 만들어 해결합니다.

비연결성은 서버와 클라이언트가 한 번 요청과 응답을 처리하면 연결을 끊는다는 특징입니다. 이 방법 또한 서버가 여러 클라이언트와 최소한의 연결을 하기 때문에 서버의 부하가 줄어드는 장점이 있습니다.

## HTTP 연결 지속에 대해서 설명해주세요

HTTP의 비연결성으로 인해 여러 데이터 예를 들면 HTML, CSS, js, IMG 등의 데이터를 요청하면 요청마다 응답하고 연결이 끊기는 상황이 발생하여 상당히 비효율적인 통신을 하게됩니다.

이러한 비효율적인 통신 방법을 피하고자 HTTP 1.1 부터 연결 지속이라는 개념이 도입되어 특정 시간 동안 서버와 클라이언트가 연결을 지속할 수 있습니다.

클라이언트 측에서 요청에 keep-alive 헤더를 담아 보내야 합니다.
